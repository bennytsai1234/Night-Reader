## 名詞對照（對話沿用台灣「直行橫列」）

- 「列」＝橫的，一列就是眼睛左到右讀的一排字。
- 「行」＝直的，上下疊起來的那條。
- 症狀用這套語言：每列最左第一個字對齊，越往右、上下的字越對不齊，
  直行格線逐列散掉，做不出「每一直行都對齊」的印刷感。

攤進字距，且每列分配點不同（標點位置/避頭尾/拉丁 run/縮排）→ 直行格點逐列漂移。

`hybrid_reader_screen.dart:799`（justify 恆開）＋

### 截圖驗證要點（假設成立時應看到）

- 漂移量向右**累積放大**（第 2 字偏一點點、第 10 字偏更多）。
- 不同列的漂移方向/幅度**不一致**（有的列鬆、有的列較正）。
- 純中文且無標點的相鄰列之間**彼此**應該幾乎對齊（同 N 同殘差）。

### 若截圖對不上，備選假設（依可能性排序）

1. **標點 ink 偏位錯覺**：全形逗號/句號 advance 是 1em 但墨水偏左下，
   格點其實對齊、視覺上像偏移——若是這個，em-grid-lock 無效，
   方向變成標點置中/擠壓（部分回到冰存案）。
2. **fallback 字型寬度微差**：正規化未涵蓋的碼位（罕字、特殊符號）落到
   fallback 字型，advance 非 1em，該字之後整列右移——檢查截圖中漂移
   起點是否總在特定字元後。
3. **縮排 placeholder 與 U+3000 寬度不一**（`layout_pump.dart:379` 寬度用
   fontSize，letterSpacing≠0 時首列偏 2×letterSpacing；使用者 letterSpacing
   預設 0，此項僅在非 0 時成立）。
4. **末行補償汙染**：`lastLineSpacingCompensation` 開啟時末列被套平均字距
   （設定值見 `AppConfig.readerLastLineSpacingCompensation`）。
5. 簡繁轉換或替換規則把某些字換成非 1em 碼位（轉換發生在正規化之後
   ——worker 路徑順序見 `reader_v2_content_transformer.dart`，值得一查）。
